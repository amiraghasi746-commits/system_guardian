# System Guardian

ابزار تک‌فایلی پایتون برای تشخیص رفتار مشکوک روی سیستم، شناسایی اتصالات ریموت فعال و اسکن شبکه‌ی محلی.

> ⚠️ **مهم:** این ابزار جایگزین آنتی‌ویروس واقعی نیست. بر اساس heuristic (رفتار مشکوک) کار می‌کنه، نه دیتابیس امضای ویروس. بخش اسکن شبکه فقط باید روی شبکه‌ای اجرا بشه که مالکش هستید یا مجوز صریح دارید؛ اسکن شبکه‌ی دیگران بدون اجازه می‌تونه از نظر قانونی مشکل‌ساز باشه.

## قابلیت‌ها

1. **Process Scanner** – بررسی پروسه‌های در حال اجرا برای رفتار/مسیر مشکوک
2. **Startup Scanner** – لیست برنامه‌های اجرای خودکار موقع بوت
3. **Connection Detector** – شناسایی اتصالات شبکه‌ی فعال (ESTABLISHED) و پورت‌های پرخطر (RDP، VNC و...)
4. **Network Scanner** – کشف سیستم‌های آنلاین و پورت‌های باز در شبکه‌ی محلی

## استفاده

bash
python3 system_guardian.py --all                    # اجرای همه‌ی اسکن‌ها
python3 system_guardian.py --local                   # فقط پروسه‌ها + startup + اتصالات
python3 system_guardian.py --network                 # فقط اسکن شبکه‌ی محلی (ساب‌نت خودکار)
python3 system_guardian.py --network --cidr 192.168.1.0/24 --ports 22,80,445,3389
python3 system_guardian.py --all --report report.json

روی ویندوز برای مشاهده‌ی کامل اتصالات و رجیستری Startup، اسکریپت رو با **Run as Administrator** اجرا کنید.

### آرگومان‌های خط فرمان

| آرگومان     | توضیح                                             |
|-------------|---------------------------------------------------|
| `--local`   | اسکن لوکال: پروسه‌ها + startup + اتصالات          |
| `--network` | اسکن شبکه‌ی محلی                                  |
| `--all`     | اجرای همه‌ی اسکن‌ها                               |
| `--cidr`    | رنج IP برای اسکن شبکه (مثال: `192.168.1.0/24`)    |
| `--ports`   | لیست پورت‌ها برای چک کردن (مثال: `22,80,443,3389`)|
| `--report`  | مسیر ذخیره‌ی خروجی JSON                           |

## خروجی گزارش (JSON)

با استفاده از `--report` می‌تونید نتایج رو در فایل JSON با ساختار زیر ذخیره کنید:

json
{
  "timestamp": "...",
  "platform": "...",
  "suspicious_processes": [...],
  "startup_entries": [...],
  "remote_connections": [...]
}

## ملاحظات قانونی و اخلاقی

- اسکن شبکه فقط روی شبکه‌های تحت مالکیت خودتان یا با مجوز صریح مجاز است.
- این ابزار برای اهداف آموزشی و مانیتورینگ شخصی طراحی شده، نه جایگزین راهکارهای امنیتی حرفه‌ای.
